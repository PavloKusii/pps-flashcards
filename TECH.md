# Технічне завдання

## Flash Cards — Пожежно-прикладний спорт

---

## 1. Загальний опис

Односторінковий веб-застосунок із набором флеш-карток на тему пожежно-прикладного спорту (ППС). Користувач переглядає картки по одній, натискає на картку щоб побачити відповідь, та переходить між картками за допомогою кнопок навігації. Прогрес відображається візуально.

**Стек:** HTML + CSS + JavaScript (без фреймворків).

**Збірка:** опціонально — Vite у zero-config режимі (`npm create vite@latest`, шаблон `vanilla`). Допускається робота без збірки — просто `index.html`, що відкривається у браузері.

---

## 2. Мета проєкту

Надати студенту практичний досвід у трьох базових технологіях веб-розробки:

| Технологія | Що відпрацьовується |
|---|---|
| **HTML** | Семантична розмітка, структура документа, доступність (`aria`-атрибути) |
| **CSS** | Позиціонування, flexbox, анімації/переходи (`transition`, `transform`), адаптивність |
| **JavaScript** | Робота з DOM, обробка подій, управління станом застосунку через змінні/об'єкти |

## 3. Контент флеш-карток

Мінімум **12 карток** із питаннями та відповідями на тему ППС. Нижче — набір для старту. Студент може розширити або замінити на власний.

| # | Питання (лицьова сторона) | Відповідь (зворотна сторона) |
|---|---|---|
| 1 | Які основні дисципліни входять до програми змагань з ППС? | Підйом по штурмовій драбині, подолання 100-метрової смуги перешкод, бойове розгортання, пожежна естафета 4×100 м, підіймання по висувній драбині |
| 2 | Яка довжина смуги перешкод у ППС? | 100 метрів |
| 3 | Скільки етапів у пожежній естафеті? | 4 етапи по 100 метрів |
| 4 | На який поверх здійснюється підйом по штурмовій драбині? | На 4-й поверх навчальної вежі |
| 5 | Який обов'язковий елемент спорядження спортсмена у ППС? | Пожежна каска, пожежно-спортивний пояс, спеціальне спортивне взуття |
| 6 | Що таке «бойове розгортання»? | Командна вправа: розгортання пожежних рукавів, їх підключення та подача води у ціль |
| 7 | Яка перешкода долається першою на 100-метровій смузі? | Двометровий дерев'яний паркан (бум) |
| 8 | Скільки пожежних рукавів несе спортсмен на смузі перешкод? | 2 рукави вагою по 5 кг кожен |
| 9 | У якому році відбулися перші змагання, що стали прообразом ППС? | 1937 рік (змагання пожежної охорони НКВС) |
| 10 | Яка міжнародна організація координує ППС? | Міжнародна спортивна федерація пожежників та рятувальників (МСФПР) |
| 11 | З якого року Україна — член МСФПР? | З 2002 року |

Картки зберігаються безпосередньо у JS-файлі як масив об'єктів:

```js
const cards = [
  { question: "Які основні дисципліни входять до програми змагань з ППС?", answer: "..." },
  // ...
];
```

## 4. Функціональні вимоги

### 4.1. Відображення картки

- На екрані видно **одну картку** за раз.
- Лицьова сторона — питання. Зворотна — відповідь.
- Натискання на картку (клік або тап) **перевертає** її з анімацією (CSS `transform: rotateY(180deg)` або аналогічна).
- Повторне натискання повертає картку на лицьову сторону.

### 4.2. Навігація

- Кнопка **«Назад»** — перехід до попередньої картки.
- Кнопка **«Вперед»** — перехід до наступної картки.
- На першій картці кнопка «Назад» заблокована (`disabled`).
- На останній картці кнопка «Вперед» заблокована (`disabled`).
- При переході картка завжди показується **лицьовою стороною** (питанням).

### 4.3. Прогрес-бар

- Горизонтальна смужка над або під карткою.
- Візуально показує яка картка зараз відкрита відносно загальної кількості.
- Текстовий індикатор: `3 / 12` (поточна / загальна).

### 4.4. Клавіатурна навігація

- Стрілка `←` — попередня картка.
- Стрілка `→` — наступна картка.
- `Пробіл` або `Enter` — перевернути картку.

## 5. Нефункціональні вимоги

### 5.1. Структура файлів

```
flash-cards/
├── index.html
├── style.css
├── app.js
└── data.js       ← масив карток (імпортується в app.js)
```

Якщо студент не використовує Vite — файли `data.js` та `app.js` підключаються через `<script>` теги. У такому випадку масив карток можна тримати прямо в `app.js`.

Якщо використовується Vite — `data.js` експортує масив через `export`, а `app.js` імпортує через `import`.

### 5.2. Адаптивність

- Застосунок коректно відображається на екранах від **360px** ширини.
- Картка займає розумну ширину (наприклад, `max-width: 480px`, по центру).
- Кнопки навігації доступні для натискання пальцем (мінімальний розмір тап-зони — 44×44px).

### 5.3. Доступність

- Картка має атрибут `role="button"` та `tabindex="0"`.
- Кнопки навігації — семантичні `<button>`.
- Стан картки (лице/зворот) має бути зрозумілий для скрінрідера (`aria-label` або `aria-live`).

### 5.4. Код

- Без використання `var` — тільки `const` та `let`.
- Без `document.write`.
- JS-логіка не має бути вписана у HTML (inline `onclick` тощо) — тільки через `addEventListener`.

## 6. Дизайн

Конкретний візуальний дизайн — на розсуд студента. Орієнтовні рекомендації:

- Фон сторінки — нейтральний (світло-сірий або приглушений).
- Картка — прямокутник з округленими кутами та легкою тінню (`box-shadow`).
- Лицьова сторона та зворотна мають візуально відрізнятися (наприклад, різним кольором фону або іконкою).
- Анімація перевертання має тривати **300–500ms** і виглядати плавно.
- Прогрес-бар — чиста CSS-реалізація (без `<progress>` або JS-бібліотек).

## 7. Критерії прийняття (Definition of Done)

Проєкт вважається завершеним коли:

1. Відкривається в браузері без помилок у консолі.
2. Відображається 12+ карток з питаннями про ППС.
3. Клік по картці перевертає її з анімацією.
4. Кнопки «Назад» / «Вперед» працюють коректно, блокуються на межах.
5. Прогрес-бар оновлюється при навігації.
6. Клавіатурна навігація працює (стрілки, пробіл).
7. Сторінка адаптивна — виглядає добре на мобільному та десктопі.
8. HTML валідний (перевірка через [validator.w3.org](https://validator.w3.org)).
9. У коді немає `var`, inline-обробників, `document.write`.

---

## 8. Бонусні завдання (необов'язково)

Для студентів, які хочуть піти далі:

- **Перемішування карток** — кнопка «Перемішати» випадково змінює порядок.
- **Лічильник «Знаю / Не знаю»** — дві кнопки після перевертання картки. В кінці — підсумок скільки карток студент знав.
- **Темна тема** — перемикач `light/dark` з використанням CSS-змінних (`--color-bg`, `--color-text` тощо).
- **Local Storage** — збереження прогресу між сесіями (яку картку переглядав останньою).
